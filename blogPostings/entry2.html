Hashing Algorithm fun April 4th, 2022 1:10 pm

Started playing around with more data structures such as hashTables, fell in love with hashing functions. Using a custom hashing function I made in one of my classes, below you can input any string, as in any word. As a result it'll spit out the hash key of that word. Just something fun to play around with. In this example I used the djb2 hashing algorithm, which uses a prime number 5381, and from this prime number preforms the following on each character in the string inputted. hash = (hash << 5) + hash + c, where hash is the current hash seed, starting from the prime 5381, and c is the value of the char in a given string. For example if we had the word "hi", the calculation would be as follows:

  1. hash = ((hash << 5) + hash) + c = ((5381 << 5) + 5381) + 'h' = (172192 + 5381) + 104 = 177677
  2. hash = ((hash << 5) + hash) + c = ((177677 << 5) + 177677) + 'i' = (5685664 + 177677) + 105 = 5863446
  3. Thus the answer is 5863446.

Of course this is only a very basic hashing algorithm that can easily be reversed, therefore it is not good for most security purposes, however it gives a good demonstration into how these algorithms work on a basic level.
